# 外投全流程自动编排 Pipeline Skill

一键串联"策略生成 → 达人筛选 → 达人排期 → 内容生成&审核"全链路，也支持任意子集组合执行。上一步的输出自动作为下一步的输入，全程无需人工传递文件。

## 调用方式

### 全链路执行（推荐）

```
/pipeline full {药品名称} {KOL资源表路径} {蒲公英价格库路径} {预算} {头腰比例} {渠道}
```

示例：
```
/pipeline full 乳果糖口服溶液 D:\data\KOL资源表.xlsx D:\data\蒲公英报价.xlsx 100万 6:4 小红书
```

### 从某一步开始（跳过已完成的步骤）

```
/pipeline from-step2 {策略报告路径} {KOL资源表路径} {蒲公英价格库路径} {预算} {头腰比例} {渠道}
/pipeline from-step3 {达人筛选列表路径} {投放周期起止}
/pipeline from-step4 {达人排期表路径} {策略报告路径}
```

### 自定义组合

```
/pipeline combo step1+step2 {药品名称} {KOL资源表路径} {蒲公英价格库路径} {预算} {头腰比例} {渠道}
/pipeline combo step1+step2+step3 {药品名称} {KOL资源表路径} {蒲公英价格库路径} {预算} {头腰比例} {渠道}
/pipeline combo step2+step3 {策略报告路径} {KOL资源表路径} {蒲公英价格库路径} {预算} {头腰比例} {渠道}
/pipeline combo step3+step4 {达人筛选列表路径} {策略报告路径}
```

## 数据传递规则

### Step 1 → Step 2

| 传递内容 | 来源 | 用途 |
|----------|------|------|
| 策略报告文件路径 | Step 1 输出的 .md 文件 | 作为 Step 2 的 campaign brief 输入 |

### Step 2 → Step 3（排期）

| 传递内容 | 来源 | 用途 |
|----------|------|------|
| 达人筛选列表 | Step 2 输出的 `{药品名}_预算筛选_{渠道}.csv` | 作为排期的达人清单输入 |
| 投放周期 | 策略报告中的预算周期（或用户指定） | 确定排期起止日期 |

### Step 3 → Step 4（内容生成）

| 传递内容 | 来源 | 用途 |
|----------|------|------|
| 达人排期列表 | Step 3 输出的 `{药品名}_达人排期.csv` | 作为 Step 4 的达人明细表输入（含发布日期） |
| 策略报告路径 | Step 1 透传（或用户指定） | 作为 Step 4 的策略报告输入 |

### 列映射（Step 2 输出 → Step 4 输入）

Step 2 的筛选列表输出格式可能与 Step 4 要求的22列格式不完全一致，pipeline 自动执行以下映射：

| Step 4 期望列 | 映射来源（Step 2 输出） | 缺失处理 |
|---------------|------------------------|----------|
| 序号 | 自动生成行号 | - |
| 来源Sheet | Step 2 标记的来源 | 填"筛选结果" |
| 平台 | 筛选时的渠道参数 | 由用户指定 |
| 自有/外采 | Step 2 标记 | 默认"外采" |
| 科室 | Step 2 中的科室/类别字段 | - |
| 达人昵称 | 达人名称字段 | - |
| ID | 达人ID字段 | - |
| 账号简介 | 账号简介字段 | - |
| 地区 | 地区字段 | 缺失填空 |
| 粉丝量(w) | 粉丝量字段 | - |
| 主页链接 | 主页链接字段 | - |
| 折扣力度 | 折扣字段 | 缺失填空 |
| 13-19 月度刊例价 | 报价相关字段 | 缺失填空 |
| 最低刊例价 | 最低价字段 | - |
| 类型标签 | Step 2 筛选标签 | - |
| 达人层级 | Step 2 层级判定 | - |

如映射后仍有必填字段缺失，pipeline 暂停并提示用户补充。

## 独立执行模式

每个 skill 也可以独立使用，不经过 pipeline：

| Skill | 独立调用 | 场景 |
|-------|----------|------|
| 策略生成 | `/strategy {药品名}` | 只需要策略报告 |
| 达人筛选 | `/kol-screen {参数}` | 已有策略报告，只需筛选达人 |
| 达人排期 | `/schedule {达人列表} {投放周期}` | 已有筛选列表，只需排期 |
| 内容生成&审核 | `/xhs {达人表} {策略报告}` | 已有达人表和策略报告 |
| 比价 | `/quote-audit {PO单} {基准库}` | 独立核验报价 |

## 异常处理与断点续跑

| 场景 | 处理 |
|------|------|
| Step 1 失败 | 提示错误，终止流程 |
| Step 2 失败 | 保留 Step 1 产出，提示从 Step 2 重跑 |
| Step 3 失败 | 保留 Step 1+2 产出，提示从 Step 3 重跑 |
| Step 4 失败 | 保留 Step 1+2+3 产出，提示从 Step 4 重跑 |
| 中途用户中断 | 已完成步骤的输出保留在 ./output/ 下，可用 from-stepN 续跑 |
| Step 2 输出列不足22列 | 自动映射 + 提示缺失字段，询问用户是否继续 |
| 用户想跳过某步 | 使用 combo 模式自由组合 |

## 比价 Skill 的定位

比价 skill（`/quote-audit`）在当前流程中为**可选附加步骤**：

- **自动触发场景**：Step 2 完成后，如果比价结果中存在"倒挂"（高于基准价）的达人 ≥ 3 位，pipeline 自动提示"是否需要执行详细比价审核？"
- **手动触发**：用户随时可以用 `/quote-audit` 独立运行
- **与 Step 2 的关系**：Step 2 内置了简单的蒲公英比价逻辑，比价 skill 提供更深入的多源核验

## 参数速查

### 全链路必填参数

| 参数 | 说明 | 示例 |
|------|------|------|
| 药品名称 | 通用名或品牌名 | 乳果糖口服溶液 |
| KOL资源表 | 含达人信息的表格 | D:\data\KOL.xlsx |
| 蒲公英价格库 | 基准报价表 | D:\data\蒲公英.xlsx |
| 预算 | 总投放预算 | 100万 |
| 头腰比例 | 头部:腰部预算分配 | 6:4 |
| 渠道 | 投放平台 | 小红书 / 抖音 |

### 可选参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 输出目录 | 自定义输出路径 | ./output/ |
| 品牌名 | 药品品牌名（如与通用名不同） | 从策略报告提取 |
| 投放周期 | 排期起止日期 | 从策略报告预算周期推算，默认当日起6个月 |
| 批量模式 | 内容生成是否分批 | 一次性全部 |
