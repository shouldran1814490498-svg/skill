# 药品社媒投放策略分析

输入一个药品名（+可选品牌/厂家），输出一份完整的社媒投放策略报告（PDF + 概览摘要）。

## 环境与工具自适应（先读这一节）

本技能可在两类环境运行，调用前先识别当前环境，再选对应的命令/工具名：

| 项 | joyclaw 平台（Linux） | 本地 Claude Code（Windows） |
|----|----------------------|----------------------------|
| Python 解释器 | `python3` | `python`（本机 `python3` 是空存根，跑不出，必须用 `python`） |
| 联网搜索 | `web_search` | `WebSearch` |
| 抓取网页 | `web_fetch` | `WebFetch` |
| 写文件 | `write` | `Write` |
| 执行命令 | `exec` | `Bash` |
| 上传文件 | `file_upload(path)` → 下载链接 | 无通用上传，降级为给出本地文件路径 |

**脚本路径**：始终用「技能所在目录」相对定位，不要写死绝对路径。
- joyclaw：脚本在 `<技能目录>/scripts/`（历史上是 `/root/.joyclaw/userUploadSkills/drug-marketing-strategy/scripts/`，以实际下发目录为准）
- 本地：脚本在本 SKILL.md 同级的 `scripts/` 目录

**PDF 渲染**：引擎已内置三级兜底，自动按序探测 `wkhtmltopdf → Chrome → Edge`，无需关心当前机器装了哪个。本地未装 wkhtmltopdf 时会自动用 Chrome/Edge 的 headless 出 PDF。

**交付方式**：
- 能上传（joyclaw）→ `file_upload(pdf_path)` 拿下载链接给用户
- 不能上传（本地）→ 直接把 PDF/MD 的本地路径告诉用户

## 完整流程

### Phase 0：先问再搜（关键规则）

用户说出药品名后，第一步不是搜索，而是先问清楚。不要等搜完才让用户知道还得补信息。

**必问的 3 个问题（用户已主动给的直接跳过、已给但不够具体的只追问细节）：**

1. **投放侧重方向？** 比如主攻减重人群还是同时打 PCOS 特殊人群、聚焦哪个适应症。
2. **有没有想强化的利益点？** 比如性价比、独家机制、品牌背书、特定人群专属。
3. **预算大概多少？** 有预算的话投放节奏能拆得更细。用户给不出/没想好 → **默认按 100 万预算、6 个月周期**出基础框架，不要在报告里写「用户未提供预算」。

**对话示例：**
> 「好的，先了解一下需求——这次投放主要侧重哪个方向？比如主攻减重人群，还是同时打 PCOS 特殊人群？有没有特别想强调的利益点？另外预算方面有数吗，有的话我可以把投放节奏拆得更细。」

**处理规则：**
- 回答模糊/没有 → 「那我基于产品特点自行分析」，继续
- 给不出预算 → 默认 100 万 / 6 个月，照常填 `budget` / `budget_duration` 字段（报告会显示）
- 回答具体 → 融入对应章节

### Phase 1：搜索药品信息

用户回复后分两步搜索（工具名按环境对照表选 `web_search`/`WebSearch`、`web_fetch`/`WebFetch`）：

**第 1 步：搜关键词**

```
# 基础信息
{drug_name} 药品 适应症 说明书
{drug_name} 作用机制 厂家 上市 价格
# 竞品
{drug_name} 竞品 对比 同类
# 社媒热度和市场认知
{drug_name} 适应症 小红书 话题 用户
```

**第 2 步：有需要时抓取详情**

搜索摘要不够用时，抓取有价值的链接。优先：
- 标题看起来有临床数据/价格信息的链接
- 医生/专家撰写的详细科普文章
- 药企官方新闻/公告

### Phase 2：填数据 → 调引擎 → 生成报告

用搜到的信息构建数据字典，按下面流程生成。

**⚠️ 关键：必须用「写 JSON 文件 → 跑加载器」模式，禁止用 heredoc（`cat << 'EOF'`），避免中文引号/长 JSON 在 shell 层被截断。**

#### 调用步骤（JSON 加载器模式 —— 推荐，彻底避免中文/Shell/heredoc 问题）

**原理**：把数据写成多个小 JSON 文件（用 write/Write 工具写 0 风险），再用 `drug_json_runner.py` 加载合并生成。

**第 1 步：把数据拆成 3-4 个 JSON 文件**（放到系统临时目录，文件名带药品名防冲突）

| 文件（示意） | 包含字段 | 建议 |
|------|---------|------|
| `rpt_{drug}_p1.json` | `drug_name` ~ `adverse_reactions` | 基础信息 |
| `rpt_{drug}_p2.json` | `compliance_tips` ~ `budget_duration` | 合规/竞品/阶段/预算 |
| `rpt_{drug}_p3.json` | `benefits` ~ `user_behavior` | 利益点/画像 |
| `rpt_{drug}_p4.json` | `kol_matrix` ~ 结尾字段 | 达人矩阵/平台/节奏/总结 |

> 💡 **为什么拆文件？** 单文件易写超长；拆成每个 60-100 行刚好。字段无关顺序——`p2.json` 里可以只放 `{"compliance_tips":[...],"competitors":[...]}`，加载器自动 merge。

**验证 JSON 语法（推荐）：** 用 `python`（本地）/`python3`（joyclaw）：
```
python -c "import json; json.load(open(r'<p1路径>', encoding='utf-8'))"
```
报错就改对应文件。

**第 2 步：跑加载器生成**（解释器按环境选 `python`/`python3`）

```bash
python <技能目录>/scripts/drug_json_runner.py \
  <p1路径> <p2路径> <p3路径> <p4路径>
```

> 输出会打印 `✅ PDF: ...pdf` 和 `MD: ...md`。若机器无任何 PDF 渲染器，会降级保留 HTML 并提示路径。

**第 3 步：交付**
- joyclaw：`file_upload(pdf_path)` 拿下载链接
- 本地：直接把 PDF/MD 路径给用户

> **备用模式（仅当 JSON 写不进去时）**：用 `gen_report_template.py`，填充其 `data` 字典后写到临时目录再跑。JSON 模式始终是首选——write 工具写 JSON 0 语法风险，出错只改片段不必重写整段 Python。

### Phase 2.5：输出前自检

生成 PDF 前做一次字段完整性自检：

- ✅ 利益点 `benefits` 每个 `point` 都配了 `user_value`（用户语言转化）
- ✅ 达人矩阵 `kol_matrix` 四类占比加起来 = 100%
- ✅ 渠道 `platforms` 占比加起来 = 100%
- ✅ `budget` / `budget_duration` 已填（用户没给就是默认 100万 / 6个月），不是空
- ✅ 合规 `compliance_tips` 覆盖了该品类最常见的高危话术
- ✅ 阶段判断 `development_stage` 与 `stage_analysis` 逻辑一致
- ✅ 节奏 `phase_tactics` 各阶段 `budget_pct` 加起来 = 100%

**发现问题 → 回头补数据再生成。** 不要让报告带着空白表格给用户。

### Phase 3：展示给用户

输出两样东西：
1. PDF 下载链接（joyclaw）或本地文件路径（本地）
2. 策略概览（5 分钟能读完的核心摘要，分 5 小节对应 PDF 内容）

**PDF 中包含（引擎自动生成）：**
- 一、药品基础档案（含竞品对比、社媒热点、合规提醒）
- 二、药品发展阶段分析
- 三、产品利益点拆解 & 社媒语言转化
- 四、投放策略框架（场景分析、达人矩阵、节奏、预算）
- 五、总结

## 关键方法论

### 利益点拆解：挖掘清单（按实际取舍）+ 三段式

下面 8 个角度是**挖掘提示清单，不是必填项**——按药品实际情况挑真正成立、有差异化的来写，**有几个写几个**。宁可 3 个扎实的利益点，也别为凑满 8 条硬编（编造的利益点既无说服力又踩合规线）。

**唯一的硬要求是三段式**：每个写出来的利益点都要走「证据 → 意义 → 用户语言」（对应字段 `evidence` / `implication` / `user_value`），缺了 `user_value` 就没法落到投放内容。**核心心法：处方药是双层决策——医生开方才是真转化，C 端内容做的是「帮患者建立信心、推动他去找医生」。**

| 角度 | 什么时候用得上 | 实战要点 | 用户语言范例 |
|------|--------------|---------|-------------|
| 机制差异 | 有独家/新机制时 | 为什么比竞品好；把专业机制翻译成「直达/精准/根源」 | 「药直接喷到病灶上，比吃药快」 |
| 权威背书 | 有指南/专家/临床数据时 | 指南推荐顺序、院士/学会、临床数据——破除「不敢信」 | 「国内外指南用了几十年，医生比你更怕不安全」 |
| 安全优势 | 用户对该品类有安全顾虑时 | 副作用小/可控；用「局部/极小剂量/可逆」对冲恐惧 | 「虽是激素但只作用在气道，全身基本不吸收」 |
| 便捷性 | 用法/渠道比竞品省事时 | 用法简单、疗程短、线上可及——降低行动门槛 | 「在家雾化就行，不用天天跑医院」 |
| 人群适配 | 有特定人群专属价值时 | 特定人群专属（儿童/孕产/老人/特殊适应症） | 「儿科一线用药，专门给孩子设计的剂量」 |
| 价格优势 | 性价比/试错门槛是卖点时 | 性价比、试错门槛、疗程装——降低决策成本 | 「启动装 99 元，比一包烟还便宜」 |
| 品牌故事 | 原研/首款/进口可讲时 | 原研/首款/进口品质——给「选它不选别人」一个理由 | 「进口原研药，医生用了二十年」 |
| 消费感 | 偏 OTC、线上可及时 | 线上可及、即买即用——把严肃医疗体验做轻 | 「京东问诊开方，到家配送」 |

> 转化时切忌堆专业名词。证据是给医生看的、意义是连接桥、**用户语言才是真正投到社媒里的话**。

### 患者旅程与转化漏斗（AISAS × 处方药双层决策）

设计投放链路时，先把患者从「不知道」到「用上药」的路走一遍。社媒时代用 **AISAS** 模型比经典 AIDA 更贴切——关键是多了 **Search（搜索）** 和 **Share（分享）** 两环：内容不只要「推到面前」，更要「铺在搜索路径上等他来搜」，并在用药后促成口碑回流。

| 漏斗阶段 | 患者在做什么 | 内容/投放任务 | 对应数据字段 |
|---------|------------|--------------|------------|
| **A 认知 Attention** | 出现症状/被科普触达，意识到「有这么个问题/新方法」 | 疾病科普、品类教育（导入期主战场） | `scenario_analysis` |
| **I 兴趣 Interest** | 想了解「这是什么、适不适合我」 | 机制通俗化、人群适配、利益点 `user_value` | `benefits` |
| **S 搜索 Search** | 主动搜「XX怎么办/XX副作用/XX安全吗」 | **在搜索结果里埋承接内容**（知乎/小红书沉淀），破除高顾虑 | `user_behavior` |
| **A 行动 Action** | 线上问诊→开方→配送，或到院开方 | 京东健康/互联网医院闭环、到院引导 | `decision_path` |
| **S 分享 Share** | 用药后晒经验、复购、推荐 | 患者真实体验、社群运营、续方提醒 | `phase_tactics` 收割期 |

**两条铁律：**
1. **处方药双层决策**——C 端漏斗的终点不是「下单」，而是「推动患者去找医生 / 建立开方信心」。**医生开方才是真转化**，C 端内容做的是把患者**送到 Search→Action 的门口**。
2. **Search 环最容易被忽略也最关键**——患者搜索时正处于高顾虑、高决策意愿的窗口，必须有内容承接。报告里 `user_behavior` 就是在还原这条搜索链，据此决定知乎/小红书要铺哪些关键词。

> 购买后会**回环**（麦肯锡 CDJ：用药→满意→推荐→复购），所以成熟期/维护期要专门经营 Share 环的复购与口碑，而非只盯拉新。

### 平台分工：按「角色」而非「热度」选平台

不同平台在漏斗里承担不同角色，**先想清楚每个平台要解决漏斗的哪一环，再分配 `platforms` 占比**：

| 平台 | 漏斗角色 | 适合内容 | 注意 |
|------|---------|---------|------|
| **抖音** | 破圈触达（A/I）| 算法去中心化、泛人群引爆；医生科普短视频、兴趣电商直播闭环 | 用户年轻化，泛人群破认知首选 |
| **小红书** | 种草 + 搜索承接（I/S）| 女性/年轻高知为主，生活化「帮家人」种草，「新一代搜索引擎」 | **严管外部导流**（加微信/询价易封号），转化靠站内 |
| **知乎** | 专业背书 + 长效沉淀（S）| 算法「倾向专家答案」，问答可长期被检索，承接高顾虑深度搜索 | 适合「XX安全吗/XX机制」这类决策前重顾虑 |
| **京东健康 / 互联网医院** | 转化闭环（Action）| 问诊→开方→配送一站式承接处方药转化 | 处方药线上转化的落点，贯穿全周期 |

> 占比无标准答案：导入期重「破圈+专业背书」（抖音+知乎），成长期向「种草+转化」倾斜（小红书+京东），按药品所处阶段和人群调整。

### 达人矩阵分层：金字塔配比随阶段移动

达人营销的底层逻辑是**信任**——小创作者（KOC/患者）胜在真实可信，大V/医生胜在权威与覆盖，要的是**组合**而非单押一层。四层角色固定，**占比随生命周期阶段移动**：

| 层级（字段） | 角色定位 | 信任来源 | 适合漏斗环 |
|------------|---------|---------|----------|
| **官媒/品牌号** `official_media` | 权威定调、品牌资产沉淀 | 官方背书 | A 认知 |
| **医生/专家 KOL** `doctor_kol` | 专业科普、破除恐惧、指南背书 | 专业权威 | I/S 兴趣+搜索 |
| **非医 KOC** `non_medical_koc` | 健康/生活方式博主，翻译成人话、场景种草 | 亲和真实 | I 兴趣 |
| **患者 KOL/真实用户** `patient_kol` | 同病相怜的真实经历，消除焦虑 | 同侪共鸣 | S 分享/口碑 |

**占比配置逻辑（经验值，非硬数据，需按药品调整）：**
- **导入期**：医生 KOL 挑大梁（科普破冰），官媒背书，KOC/患者占比低 —— 如医生 40%+/官媒 15%+。
- **成长期**：医生 KOL 适度下调、**非医 KOC 与患者 KOL 上量**做扩面与口碑裂变 —— 如医生 40%/非医 KOC 30%/患者 20%/官媒 10%。
- **成熟/维护期**：患者 KOL 与私域口碑权重升高，主攻复购与场景下沉。

> ⚠️ 以上占比是行业惯例与样例数据的经验区间，**不是权威硬指标**。落地时按品类（严肃🟢/偏OTC🟡/罕见🟠）、阶段、预算微调，并确保四层占比合计 = 100%（见 Phase 2.5 自检）。

### 药品生命周期五段论与营销分析框架

分析时**必须先判断药品当前处于哪个生命周期阶段**，再按对应阶段输出策略。每个阶段从 8 维度展开：阶段特征 / 核心营销目标 / KPI 侧重 / 主力营销动作 / 渠道策略 / 内容策略 / 合规边界 / 典型场景。

---

#### 一、开发期/上市前（研发~获批准备期）

| 维度 | 内容 |
|------|------|
| **阶段特征** | 尚未获批；临床/审批阶段；零销售；高研发投入；仅学术圈认知 |
| **核心目标** | 学术预热，让医生/专家了解新治疗可能性，建立「期待感」 |
| **KPI 侧重** | 不适用传统投放 → 学术会议参与度、专家共识引用率、预注册患者数据 |
| **主力动作** | 学术会议发布临床数据；绑定顶尖 PI；选高影响力试验基地；疾病教育（不涉品牌） |
| **渠道策略** | 院内学术推广为主；线上做疾病科普不做品牌暗推 |
| **内容策略** | 疾病新进展、临床数据解读、未被满足的需求故事 |
| **合规边界** | ❌ 提前宣传未获批适应症；❌ 暗示"即将上市"当卖点；✅ 疾病科普教育 |
| **典型** | 某 PD-1 获批前 18 个月在肺癌学术圈预热临床数据 |

---

#### 二、导入期/上市初期（刚获批~上市6-12个月）

| 维度 | 内容 |
|------|------|
| **阶段特征** | 刚获批；认知度极低(<5%)；销量小利润负；竞争者极少；教育成本高。**核心一个"短"字**——尽快冲过认知鸿沟 |
| **核心目标** | **品类教育**：让目标人群知道"有这个药/品类存在"；首方故事建立初始信任 |
| **KPI 侧重** | 品牌/品类关键词搜索增长、医生处方尝试率、媒体声量(SOV)、认知度前后对比 |
| **主力动作** | KOL 科普破冰（医生/官媒/权威背书）；第一方临床数据可视化；种子用户试用；电商搭闭环(问诊-开方-配送) |
| **渠道策略** | 抖音/知乎/视频号做医生深度科普；小红书做"帮家人"种草；京东健康站内承接；学术会议继续教育 |
| **内容策略** | 90% 品类教育 + 10% 品牌露出。医生侧"这个东西存在"，患者侧"不是你没用，是有新方法" |
| **合规边界** | ❌ 承诺疗效；❌ 比竞品排名；❌ "秒杀/神药/x天见效"；✅ 临床数据解读/指南推荐/医生观点/脱敏真实案例；✅ 患教内容可不说药名只讲方法 |
| **典型** | 乐速克（伐尼克兰）——戒烟品类认知<5%，先让烟民知道有戒烟药这种选择 |
| **预算建议** | 虽亏损也要保持投入。**这阶段不做 C 端教育就没有成长期**，预算侧重医生 KOL + 品类科普 |

---

#### 三、成长期/快速增长（认知度5%-30%+，销量爬坡）

| 维度 | 内容 |
|------|------|
| **阶段特征** | 品类认知初步建立；销量迅速增长；成本下降利润上升；竞品入场。**核心一个"快"字**——快速占位+拓圈 |
| **核心目标** | **品牌占位**：品牌=品类代名词；**扩面**：核心人群→泛人群；建立差异化打击竞品 |
| **KPI 侧重** | 品牌声量占比(SOV)、电商转化率、首处占比、NPS、内容互动率、投放 ROI |
| **主力动作** | 达人矩阵扩面(医生→非医KOC→患者KOL)；套餐/疗程装提客单价；渠道全面铺开；差异化定位强化 |
| **渠道策略** | 抖音+小红书扩泛人群；知乎沉淀口碑；京东直接转化；私域社群+企微；下沉渠道试点 |
| **内容策略** | 60% 品牌占位 + 30% 品类巩固 + 10% 口碑裂变。医生KOL 40%/非医KOC 30%/患者KOL 20%/官媒 10% |
| **合规边界** | ❌ 竞品攻击(哪怕暗示)；❌ 打折促销暗示(处方药)；✅ 差异化机制/指南推荐顺序/临床数据更新 |
| **典型** | 达卫可（莱博雷生）在失眠品类从"褪黑素"转向"真失眠药" |
| **预算建议** | 预算达高峰，舍得在头部 KOL 抢心智 |

---

#### 四、成熟期/稳定期（认知30%+，增长趋缓）

| 维度 | 内容 |
|------|------|
| **阶段特征** | 销量最大+成本最低+利润最高+竞争最激烈。**核心一个"争"字**——防守+挖掘 |
| **核心目标** | **维护份额**：守核心阵地；**场景下沉**：找新人群/新场景；防竞品挖墙脚 |
| **KPI 侧重** | 市场份额、复购/续处方率、竞品抢量监控、私域留存率 |
| **主力动作** | 价格优惠/患者援助+渠道深耕+教育性促销；找新细分人群/地域/年龄；新剂型/新规格 |
| **渠道策略** | 存量维护+新渠道(低线/基层)；私域运营(社群/复诊提醒)；医生继续教育防品牌老化 |
| **内容策略** | 40% 品牌维护 + 30% 场景下沉 + 20% 口碑经营 + 10% 竞品防御 |
| **合规边界** | ❌ 全面"打击竞品"话术；⚠️ 降价宣传敏感；✅ 新适应症/新剂型可做新一轮科普 |
| **典型** | 某降压药找到"职场高压人群"新场景拉第二波增长 |
| **预算建议** | 转向精准投放，重点保核心市场 |

---

#### 五、维护期/延长期（增长放缓/下滑）

| 维度 | 内容 |
|------|------|
| **阶段特征** | 销量下滑；利润下降；竞品围剿；品牌老化 |
| **核心目标** | **延缓衰退**：新适应症/新剂型/新市场；**榨取剩余价值** |
| **KPI 侧重** | 利润率、患者留存、渠道覆盖率 |
| **主力动作** | 新适应症申报；OTC化/双跨；新剂型(缓释/咀嚼/口服液)；基层下沉 |
| **渠道策略** | 收缩到最赚钱核心市场；基层医疗覆盖 |
| **内容策略** | 新适应症教育/OTC 场景教育；老适应症维持最小声量 |
| **合规边界** | ❌ 虚假"升级/换代"；⚠️ 新适应症需审批不能提前 |
| **典型** | 阿司匹林：急救→低剂量预防→复合制剂 |
| **预算建议** | 最低预算，仅核心阵地维护 |

---

#### 跨阶段通用规则

**1. 处方药 C 端营销分类（决定合规边界宽窄，按品类先归类再定红线）：**

| 分类 | 特征 | C 端边界（红线） |
|------|------|---------|
| 🟢 严肃处方药 | 肿瘤/免疫/重症 | 只做患者疾病教育（❌ 不涉及药品品牌）+ 依从性管理 + 临床招募 |
| 🟡 偏 OTC 属性 | 慢病/皮肤/儿科 | 品牌背书 + 场景教育 + 口碑内容，**社媒投放主战场**，边界相对宽 |
| 🟠 罕见病 | 患者极少/诊疗集中 | 患者社群 + 诊疗路径科普 + 全链条支持，边界最宽 |

**2. 阶段判断参考（销售增长率法）：** <10%→导入期 | >10%→成长期 | 0.1%~10%→成熟期 | 负增长→维护期

**3. 多竞品/多适应症的交叉处理：**

| 场景 | 处理方式 |
|------|---------|
| 本品成长 / 核心竞品成熟 | 成长药全面进攻(扩面抢心智)，对成熟竞品做差异化攻击 |
| 本品导入 / 竞品已成熟 | 不正面硬刚，打品类增量用户不争存量；内容侧重品类教育而非竞品对比 |
| 同药不同适应症分处不同阶段 | 按预算权重分配资源，以主要适应症当前阶段为主策略 |
| 多竞品各处不同阶段 | 盯最近(竞争最直接)的那个打，忽略远期 |

**4. 核心原则（所有阶段通用）：**
- 利益点拆解必须有**用户语言转化**
- 始终记住**处方药双层决策**（医生开方才是真转化，C 端做的是帮患者找医生/建立信心）
- 合规不是成本，**是护身符**

在 Claude Code 里依次执行：

```
/plugin marketplace add Alvinxie323716/drug-marketing-strategy

```

### 二、开启自动更新（重要，需手动开一次）

第三方市场默认**关闭**自动更新，需使用者手动开启一次：

1. 输入 `/plugin` 打开菜单
2. 进入 **Marketplaces** 标签
3. 选中 `alvinxie-skills`
4. 选择 **Enable auto-update**

开启后，每次启动 Claude Code 会自动拉取最新版，并提示 `/reload-plugins` 应用更新。

> 若不想开自动更新，也可随时手动更新：
> `/plugin marketplace update alvinxie-skills` 然后 `/reload-plugins`

### 三、发布新版（面向作者）

技能内容改动后，只需推送到 GitHub：

```
git add -A && git commit -m "更新说明" && git push
```

`plugin.json` 中 `version` 字段留空，Claude Code 会按每个 commit 自动判定为新版本——**无需手动改版本号**。已开启自动更新的使用者下次启动即可收到。
