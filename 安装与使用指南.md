# 药品社媒外投技能包 — 安装与使用指南

## 一、安装

### 方法一：让 Claude Code 帮你安装（最简单）

打开 Claude Code，直接对它说：

> 帮我从 https://github.com/shouldran1814490498-svg/skill 下载所有 skill 文件到本地 .claude/commands/ 目录

它会自动完成 clone、文件复制、目录创建等所有操作。

### 方法二：手动安装

1. 下载仓库（任选一种）：
   - 在终端执行：`git clone https://github.com/shouldran1814490498-svg/skill.git`
   - 或直接在 GitHub 页面下载 ZIP 解压

2. 找到你的 Claude Code 配置目录：
   - **Windows**：`C:\Users\你的用户名\.claude\commands\`
   - **Mac/Linux**：`~/.claude/commands/`
   - 如果 `commands` 文件夹不存在，手动创建一个

3. 复制文件（按以下对应关系）：

   | 仓库里的文件 | 复制到 commands/ 下并改名为 |
   |-------------|--------------------------|
   | `外投策略生成skill/SKILL.md` | `strategy.md` |
   | `外投达人筛选skill/医生达人筛选看板_Skill.md` | `kol-screen.md` |
   | `外投排期skill/SKILL.md` | `schedule.md` |
   | `外投小红书内容生成&审核skill/SKILL.md` | `xhs.md` |
   | `外投比价skill/SKILL.md` | `quote-audit.md` |
   | `pipeline/pipeline.md` | `pipeline.md` |

4. 复制脚本目录：
   - 把 `外投策略生成skill/scripts/` 整个文件夹复制到 `.claude/commands/scripts/`

### 方法三：配置自动更新（推荐所有人开启）

安装完成后，对 Claude Code 说：

> 帮我配置 skill 自动同步，仓库地址是 https://github.com/shouldran1814490498-svg/skill

这会创建一个同步脚本 + hook，每次你使用 Claude Code 时自动检查 GitHub 是否有更新，有则自动拉取。冷却时间1小时，不影响使用速度。

### 验证安装成功

安装后在 Claude Code 中随便输入一个命令试试：
- 输入 `/strategy` → 看到策略生成的 skill 说明
- 输入 `/pipeline` → 看到全流程编排的 skill 说明

看到说明文字就是成功了。

## 二、技能概览

```
┌────────────┐     ┌────────────┐     ┌────────────┐     ┌────────────────┐
│  策略生成   │ ──→ │  达人筛选   │ ──→ │  达人排期   │ ──→ │ 内容生成&审核   │
└────────────┘     └────────────┘     └────────────┘     └────────────────┘
                          │
                          └──→ 比价核验（可选）
```

| 技能 | 一句话说明 | 触发命令 |
|------|-----------|----------|
| 策略生成 | 输入药品名 → 输出完整投放策略报告 | `/strategy` |

| 流水线编排 | 串联以上技能，自动传递数据 | `/pipeline` |

## 三、单独使用

### 1. 策略生成

生成某药品的完整社媒投放策略报告。

```
/strategy 乳果糖口服溶液
```

**输入**：药品名称（必填），品牌名/厂商（可选）

**输出**：`{药品名}_投放策略报告.md`（同时生成 PDF 和 HTML），包含：
- 药品基础档案 & 合规红线
- 利益点拆解 & 社媒语言转化
- 竞品格局分析
- 达人矩阵 & 内容方向
- 投放节奏建议

---

### 2. 达人筛选

根据策略报告和 KOL 资源表，自动筛选达人、分配预算、生成可视化看板。

```
/kol-screen {策略报告} {KOL资源表} {蒲公英价格库} {预算} {头腰比} {渠道}
```

**示例**：
```
/kol-screen D:\data\策略报告.md D:\data\KOL资源表.xlsx D:\data\蒲公英报价.xlsx 100万 6:4 小红书
```

**输入**：

| 参数 | 说明 |
|------|------|
| 策略报告 | 来自策略生成的 .md 文件，或你自己准备的 brief |
| KOL资源表 | 含达人信息的表格（Joyspace链接 或 本地Excel） |
| 蒲公英价格库 | 蒲公英平台报价表（Joyspace链接 或 本地Excel） |
| 预算 | 总投放预算，如"100万" |
| 头腰比 | 头部:腰部预算分配，如"6:4" |
| 渠道 | 小红书 / 抖音 |

> 如果不提供 KOL 资源表和蒲公英价格库，会自动使用内置的默认 Joyspace 数据源。

**输出**：
- `{药品名}_预算筛选_{渠道}.csv` — 筛选后的达人列表
- `{药品名}_投放看板.html` — ECharts 可视化看板（浏览器打开）
- `{药品名}_比价结果.csv` — 与蒲公英报价对比

---

### 3. 达人排期

为筛选后的达人分配具体的发布日期。

```
/schedule {达人筛选列表} {投放周期}
```

**示例**：
```
/schedule ./output/step2/乳果糖_预算筛选_小红书.csv 2026-07-01至2026-12-31
```

**输入**：

| 参数 | 说明 |
|------|------|
| 达人筛选列表 | 来自达人筛选的 CSV/XLSX 文件 |
| 投放周期 | 起止日期，如"2026-07-01至2026-12-31"或"今天起6个月" |

**排期逻辑**：
- 均匀分散到整个周期，避免集中爆发
- 头部达人优先安排在电商节点（双11预热、双12）
- 结合健康日、科普日等节点加密相关达人
- 避开周末，每日最多3人

**输出**：
- `{药品名}_达人排期.csv` — 原表 + 新增"发布日期"列
- `{药品名}_排期报告.html` — 排期逻辑说明 + 月度分布图表

---

### 4. 内容生成 & 审核

为每位达人生成个性化小红书笔记，自动执行 7 维合规审核。

```
/xhs {达人明细表} {投放策略报告}
```

**示例**：
```
/xhs D:\data\达人明细.xlsx D:\data\乳果糖口服溶液_投放策略报告.md
/xhs D:\data\达人明细.csv D:\data\乳果糖口服溶液_投放策略报告.md
```

**输入**：

| 参数 | 说明 |
|------|------|
| 达人明细表 | 22列标准格式的 .xlsx 或 .csv 文件（也接受筛选/排期输出的格式） |
| 投放策略报告 | 来自策略生成的 .md 文件 |

**输出**：在原表基础上新增 23-29 列：

| 新增列 | 内容 |
|--------|------|
| 第23列 | 分配角度 |
| 第24列 | 笔记标题 |
| 第25列 | 笔记正文（原始版） |
| 第26列 | 话题标签 |
| 第27列 | 封面建议 |
| 第28列 | 合规审核结果（评分+问题+风险） |
| 第29列 | 修正后正文（合规通过则填"无需修改"） |

---

### 5. 比价核验

对比供应商 PO 报价与基准价格库，输出风险等级。

```
/quote-audit {PO报价单} {基准价格库}
```

**示例**：
```
/quote-audit D:\data\供应商报价.xlsx D:\data\蒲公英基准价.xlsx
```

**输入**：供应商PO单 + 基准价格（蒲公英/自定义），支持 Excel、CSV、在线文档

**输出**：结构化 Markdown 审核报告，含：
- 逐条比对结果（报价/基准价/比值/风险等级）
- 总体风险评级
- 异常项标记 & 建议动作

## 四、组合使用

你可以把任意几个技能串起来用。上一步的输出文件直接作为下一步的输入。

### 示例 1：策略 + 筛选 + 排期（不需要内容）

```
/strategy 乳果糖口服溶液
/kol-screen ./output/step1/乳果糖_投放策略报告.md D:\KOL.xlsx D:\蒲公英.xlsx 100万 6:4 小红书
/schedule ./output/step2/乳果糖_预算筛选_小红书.csv 2026-07-01至2026-12-31
```

### 示例 2：已有策略和达人表，直接生成内容

```
/xhs D:\data\达人明细.xlsx D:\data\策略报告.md
```

### 示例 3：筛选 + 比价（聚焦价格审核）

```
/kol-screen D:\策略报告.md D:\KOL.xlsx D:\蒲公英.xlsx 100万 6:4 小红书
/quote-audit ./output/step2/乳果糖_预算筛选_小红书.csv D:\蒲公英.xlsx
```

## 五、全链路一键执行（Pipeline）

使用流水线编排，从药品名开始一路到最终内容输出，全程无需手动传递文件。

### 全链路执行

```
/pipeline full {药品名} {KOL资源表} {蒲公英价格库} {预算} {头腰比} {渠道}
```

**示例**：
```
/pipeline full 乳果糖口服溶液 D:\data\KOL资源表.xlsx D:\data\蒲公英报价.xlsx 100万 6:4 小红书
```

**执行过程**：
```
Step 1: 策略生成 ── 输出策略报告
         │
Step 2: 达人筛选 ── 输出筛选列表 + 看板
         │
Step 3: 达人排期 ── 输出排期表 + 排期报告
         │
Step 4: 内容生成&审核 ── 输出完整内容表
         │
      汇总报告 (pipeline_summary.md)
```

全程自动，你只需等待最终结果。

---

### 从中间步骤开始（已有部分产出）

```
# 已有策略报告，从筛选开始
/pipeline from-step2 D:\策略报告.md D:\KOL.xlsx D:\蒲公英.xlsx 100万 6:4 小红书

# 已有筛选列表，从排期开始
/pipeline from-step3 ./output/step2/筛选结果.csv 2026-07-01至2026-12-31

# 已有排期表，从内容生成开始
/pipeline from-step4 ./output/step2/达人排期.csv D:\策略报告.md
```

## 七、常见问题

**Q: 怎么安装？**
A: 最简单的方式是打开 Claude Code 直接说"帮我从 https://github.com/shouldran1814490498-svg/skill 下载所有 skill 到本地"。

**Q: 怎么更新？**
A: 配置了自动同步后无需操作。没配置的话，对 Claude Code 说"帮我更新 skill"即可。

**Q: 达人表必须是 22 列吗？**
A: 内容生成 skill 要求 22 列标准格式。但如果你是从筛选 skill 产出的表格直接传入，pipeline 会自动做列映射适配。

**Q: 支持 CSV 还是 Excel？**
A: 都支持。`.xlsx` 和 `.csv`（自动检测编码 UTF-8/GBK）均可。

**Q: 策略报告一定要用策略生成 skill 产出的吗？**
A: 不一定。只要你的报告包含药品档案、利益点拆解、合规红线、达人矩阵等标准章节，就可以直接作为输入。

**Q: 全链路跑到一半失败了怎么办？**
A: 已完成步骤的输出保留在 `./output/` 下，用 `/pipeline from-step2`（或 3、4）从断点继续。

**Q: 不同人安装的版本会不一样吗？**
A: 不会。所有人从同一个 GitHub 仓库安装，配置了自动同步后始终保持最新。

**Q: 数据源（KOL资源表、蒲公英报价库）从哪里来？**
A: 达人筛选 skill 内置了默认的 Joyspace 在线数据源。你也可以提供自己的本地 Excel 或其他 Joyspace 链接。
