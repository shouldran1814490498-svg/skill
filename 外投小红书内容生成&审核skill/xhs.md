# 小红书达人笔记生成与审核

根据**投放策略报告**（提供药品信息、利益点、合规红线等）和**达人明细表**（提供达人画像），为每位达人生成符合其身份的小红书笔记内容（确保不雷同），生成完成后自动执行合规审核，最终输出可发布的内容。全程无需人工干预。

本 skill 为通用模板，适用于任何药品——具体药品信息完全由输入的策略报告决定。

## 参数

用户提供：
1. **达人明细表**（必填）：表格链接或路径，支持 .xlsx / .csv 格式
2. **投放策略报告**（必填）：策略报告文件路径

调用格式：`/xhs {达人明细表路径} {投放策略报告路径}`

示例：
- `/xhs D:\data\达人明细.xlsx D:\data\福元_乳果糖口服溶液_投放策略报告.md`
- `/xhs D:\data\达人明细.csv D:\data\福元_乳果糖口服溶液_投放策略报告.md`

## 执行流程总览

```
读取策略报告 → 读取达人表格 → 解析画像 → 分配角度（基于策略利益点） → 生成内容（基于策略用户语言） → 去重自检 → [自动] 合规审核（基于策略合规红线） → 生成修正版 → 写回表格 → 输出汇总
```

**全程自动执行，用户只需提供表格和策略报告，最终拿到完整结果。**

## 第零阶段：读取投放策略报告

在处理达人明细表之前，先读取并解析投放策略报告，提取以下关键信息供后续阶段使用。

**重要原则：本 skill 不内置任何药品信息。所有药品名称、适应症、利益点、合规红线、竞品清单等内容，均从用户本次提供的投放策略报告中动态读取。不同药品的策略报告格式结构相同，但具体内容完全不同。**

### 从策略报告中提取的信息

| 提取项 | 来源章节 | 用途 |
|--------|----------|------|
| 药品基础档案 | 一、药品基础档案 | 了解药品通用名、适应症、作用机制、不良反应，确保内容科学准确 |
| 社媒传播合规红线 | 一、合规提醒 | 作为生成和审核阶段的硬性约束条件 |
| 产品利益点 & 用户语言 | 三、利益点拆解 | 生成内容时直接采用策略中的"用户语言"表述，确保一致性 |
| 达人矩阵占比 | 四、4.2 达人矩阵 | 分配角度时参考各类达人的角色定位 |
| 内容方向 & 利益点匹配 | 四、4.4 内容方向表 | 分配角度时匹配利益点与平台 |
| 竞品格局 | 一、竞品格局 | 审核时判断是否涉及竞品贬低 |
| 合规注意事项 | 四、合规注意事项 | 审核阶段的专项检查依据 |

### 策略报告格式要求

策略报告为 Markdown 文件，包含以下标准章节（内容因药品不同而变化）：

```
一、药品基础档案（含合规提醒、竞品格局）
二、药品发展阶段分析
三、产品利益点拆解 & 社媒语言转化
四、投放策略框架（含达人矩阵、投放节奏、内容方向、合规注意事项）
五、总结
```

## 第一阶段：内容生成

### 文件格式处理

| 格式 | 读取方式 | 注意事项 |
|------|----------|----------|
| .xlsx | 直接读取第一个工作表 | 多Sheet时取第一个 |
| .csv | 自动检测编码和分隔符 | 见下方规则 |

**CSV 读取规则：**
1. **编码检测优先级**：UTF-8 with BOM → UTF-8 → GBK/GB2312 → 自动探测
2. **分隔符检测**：逗号 `,` → 制表符 `\t` → 分号 `;`（取首行分割后列数=22的分隔符）
3. **表头校验**：首行必须匹配22列表头（允许列名前后有空格，自动trim）
4. **空行处理**：跳过全空行
5. **字段内换行**：支持双引号包裹的多行字段（RFC 4180标准）
6. **写回格式**：输出文件与输入格式一致（输入csv则输出csv，输入xlsx则输出xlsx）

### 输入表格格式

表头固定为以下 22 列：

| 列号 | 字段 | 生成时用途 |
|------|------|-----------|
| 1 | 序号 | 排序 |
| 2 | 来源Sheet | - |
| 3 | 平台 | 决定内容风格（小红书 vs 抖音） |
| 4 | 自有/外采 | - |
| 5 | 科室 | 决定专业方向和内容角度 |
| 6 | 达人昵称 | 人设匹配 |
| 7 | ID | - |
| 8 | 账号简介 | **核心字段**：提取专业方向、擅长领域、医院/机构、粉丝画像 |
| 9 | 地区 | 可融入地域相关内容 |
| 10 | 粉丝量(w) | 决定内容深度和调性 |
| 11 | 主页链接 | - |
| 12 | 折扣力度 | - |
| 13-19 | 月度刊例价 | - |
| 20 | 最低刊例价 | - |
| 21 | 类型标签 | **核心字段**：决定内容方向（如 核心-XX科 / 泛科普/健康 等） |
| 22 | 达人层级 | 决定内容深度（头部=深度专业 / 腰部=实用接地气） |

### Step 1：解析达人画像

读取表格，为每位达人提取关键信息：

```
达人画像 = {
  序号, 昵称, 平台, 科室, 账号简介, 地区,
  粉丝量, 类型标签, 达人层级
}
```

### Step 2：分配内容角度（去重核心）

#### 内容角度池（动态生成）

内容角度池不再写死，而是基于投放策略报告动态构建：

**构建来源：**
1. 从策略报告「三、产品利益点拆解」提取所有利益点作为角度种子
2. 从策略报告「四、4.3 达人投放策略」提取各类达人的内容方向
3. 从策略报告「四、4.4 内容方向&利益点匹配表」提取内容方向列表
4. 从药品适应症、作用机制中派生通用角度（机制科普、适应症科普、副作用管理、患者教育、误区澄清、联合治疗/生活管理、特殊人群、案例分享、就医指导等）

**角度池构建规则：**
- 每个策略利益点至少对应1个内容角度
- 每个内容方向（策略4.4表）至少对应1个角度
- 通用角度根据药品特性取舍（如药品无中医关联则不设中西结合角度）
- 最终角度池数量应 ≥ 达人数量（不够时执行扩展策略）

**示例**（以乳果糖为例，实际按策略报告动态生成）：

| 角度类别 | 具体话题（由策略报告决定） | 适合达人类型 |
|----------|---------------------------|-------------|
| 机制科普 | 该药品的作用原理（通俗+专业版） | 头部专科专家 |
| 适应症科普 | 什么情况该用/不该用 | 相关科室主任/教授 |
| 副作用管理 | 常见不良反应应对方案 | 实操型医生 |
| 利益点1场景 | 来自策略利益点1的用户语言 | 按策略达人矩阵匹配 |
| 利益点2场景 | 来自策略利益点2的用户语言 | 按策略达人矩阵匹配 |
| 利益点N场景 | ... | ... |
| 患者教育 | 用药期间生活注意事项 | 腰部科普达人 |
| 误区澄清 | 常见认知误区纠偏（来自策略合规红线的反面） | 学术背景强的医生 |
| 真实经历分享 | 典型改善路径（脱敏处理） | 患者KOL/真实用户 |
| 特殊人群 | 策略中提及的特殊人群场景 | 专科细分方向医生 |
| 竞品/品类对比科普 | 品类差异化科普（不贬损竞品） | 科普定位账号 |
| 季节/场景变体 | 不同场景用药注意事项 | 泛科普健康达人 |

#### 角度不够时的扩展策略（达人数 > 角度池数量）

1. **细分策略利益点**：如"温和不依赖"拆为"不刺激肠壁""不形成依赖""长期用药无耐受"
2. **组合角度**：如"误区澄清 × 孕产" = "孕期便秘用药常见误区"
3. **平台差异化**：同一角度在小红书和抖音各用一次（形式不同不算雷同）
4. **季节/场景变体**：如"夏季肠道养护""秋冬便秘高发期应对"
5. **按策略脚本示意扩展**：参考策略报告中各类达人的脚本示意，派生更多角度

#### 分配规则

1. **同一科室不重复角度**：同科室达人每人分配不同角度
2. **匹配专业背景**：根据账号简介中的擅长领域匹配最相关角度
3. **层级匹配深度**：头部→深度专业内容；腰部→实用易懂内容
4. **达人类型匹配策略矩阵**：参照策略报告的达人矩阵（医生KOL/非医KOC/患者KOL/品牌号），为不同类型达人优先分配对应角色定位的角度
5. **中医背景特殊处理**：昵称/简介含"中医"→优先分配中西结合角度（如策略涉及）
6. **抖音平台调整**：抖音达人内容更口语化，适合短视频脚本风格
7. **重复达人处理**：同一达人出现多次（不同来源），只生成一条内容，其余行标记"同达人-见第X行"

### Step 3：按人设生成内容

根据策略报告中的达人矩阵和利益点，结合达人画像匹配人设风格：

| 达人特征 | 内容风格 | 语气 | 品牌处理 |
|----------|----------|------|----------|
| 头部+主任/教授 | 权威科普，引用指南/数据 | 专业严谨，第三人称 | 只提品类通用名，不提品牌名 |
| 头部+硕士/博士 | 学术科普，机制解析 | 学术通俗化，有深度 | 只提品类 |
| 腰部+主治医师 | 实用指导，临床经验 | 亲切专业，第一人称 | 模糊化（用品类泛称） |
| 腰部+泛科普 | 生活化科普，场景切入 | 朋友聊天风，轻松 | 绝不提品牌 |
| 非医KOC/博主 | 场景种草，生活经验 | 真实亲切，共情感强 | 不提品牌 |
| 患者KOL/真实用户 | 第一人称经历分享 | 同病相怜，温暖共情 | 不提品牌，模糊化 |
| 品牌号/官媒 | 正式科普，品牌定调 | 专业权威但不硬推 | 可提品牌但不做疗效承诺 |
| 中医背景 | 中西医结合视角 | 温和稳重 | 只提品类 |
| 跨科室医生 | 跨科视角关联科普 | 科普关联 | 不提品牌 |

**生成时的策略报告引用规则：**

1. **用户语言优先**：正文表述必须基于策略报告「三、利益点拆解」中的"用户语言"字段，不得自行编造卖点
2. **合规红线内化**：策略报告「一、合规提醒」和「四、合规注意事项」中的 ❌ 表述绝对不能出现，✅ 替代表述作为安全参考
3. **利益点覆盖**：每篇笔记至少关联策略报告中的1-2个核心利益点
4. **竞品处理**：参照策略报告中的竞品格局，仅做品类对比科普，禁止贬损或排名
5. **适应症边界**：严格在策略报告标注的适应症范围内，不扩展到非适应症功效

### Step 4：去重自检

生成全部内容后，进行去重自检：

1. **标题去重**：所有标题不得有超过3个相同关键词
2. **开头去重**：每篇开头的切入方式不同（提问/数据/场景/案例/误区/对话/热搜）
3. **结构去重**：混合使用不同结构（清单体/故事体/问答体/科普体/对比体/时间线体/红黑榜体）
4. **话题去重**：每篇的核心话题点不重复
5. **如发现雷同**：自动调整角度重新生成

| 格式 | 图文笔记，分段+emoji | 短视频口播脚本 |

## 第二阶段：合规审核（生成完成后自动执行）

**无需用户发消息触发，第一阶段完成后立即进入本阶段。**

### Step 5：加载合规知识库

审核前加载以下全部规则，逐条比对。

#### 一、小红书平台社区规范

**社区公约核心原则**：

| 原则 | 要求 | 违反后果 |
|------|------|----------|
| 真实性 | 分享真实经历，不编造虚假体验、不伪造身份 | 限流/删除/封号 |
| 原创性 | 内容为原创，不搬运/洗稿/抄袭 | 删除+降权 |
| 善意表达 | 不攻击、不诋毁、不引战、不造谣 | 禁言/封号 |
| 科学性 | 涉及健康/医疗/科学话题需有据可查 | 限流/删除 |

**内容发布红线（触碰即删除/封号）**：

| 类别 | 具体禁止内容 |
|------|-------------|
| 违法信息 | 涉政敏感、暴力恐怖、赌博诈骗、毒品 |
| 侵权内容 | 盗用他人图文/视频、侵犯肖像权/隐私权 |
| 虚假信息 | 编造医疗案例、伪造检验报告、虚构用户评价 |
| 低俗色情 | 软色情、擦边暗示、未成年人不当内容 |
| 站外引流 | 留微信号/QQ/手机号/外部链接/暗语引导 |
| 交易引导 | 未经平台的商品交易引导、私下成交 |
| 恶意营销 | 刷量/互赞/养号/批量发布同质内容 |

**限流触发条件**：

| 触发行为 | 限流程度 | 恢复难度 |
|----------|----------|----------|
| 笔记含违禁词/敏感词 | 该篇限流 | 修改后可恢复 |
| 疑似广告未标注 | 该篇限流 | 标注"合作"后恢复 |
| 频繁发布同质内容 | 账号降权 | 需等待周期 |
| 被举报+判定违规 | 该篇删除 | 申诉 |
| 多次违规累积 | 账号全面限流 | 极难恢复 |
| 涉及医疗违规 | 即时限流+删除 | 需修改+申诉 |

**违禁词库（高频踩雷）**：

- 绝对化用语：最、第一、唯一、顶级、极致、绝对、100%、万能、永久、根治、史上最强、独一无二、全网最低、No.1
- 虚假承诺词：立竿见影、一次见效、药到病除、包治、保证有效、无效退款、零副作用、纯天然无害
- 医疗违规词：治愈、根治、痊愈、再也不会复发、替代手术、替代药物、比XX药好、不用去医院
- 引流违规词：私我、滴滴、评论区找我、主页联系方式、加V、咨询链接在...

#### 二、小红书医疗健康内容专项规范

**内容分级管理**：

| 内容级别 | 定义 | 平台态度 | 发布要求 |
|----------|------|----------|----------|
| 健康科普 | 通用健康知识、养生常识 | 鼓励 | 需标注"仅供参考" |
| 疾病教育 | 特定疾病的科普知识 | 允许 | 需有专业资质佐证 |
| 诊疗经验 | 就医经历、治疗过程分享 | 有条件允许 | 不得推荐具体机构/医生做治疗引导 |
| 药品信息 | 涉及具体药品的内容 | 严格审核 | 不得构成药品广告 |
| 处方药推广 | 直接或变相推广处方药 | **禁止** | 一律删除+处罚 |

**医疗健康类账号要求**：

| 账号类型 | 平台要求 | 内容权限 |
|----------|----------|----------|
| 认证医生 | 需完成专业认证（执业证+工作证明） | 可发布专业诊疗知识 |
| 未认证但声称医生 | 高风险，平台会重点审核 | 不得以医生身份做推荐 |
| 健康博主 | 无需认证 | 仅限通用健康科普 |
| 患者分享 | 无需认证 | 可分享经历但不得做医学建议 |

**笔记审核重点**：

| 审核项 | 通过标准 | 不通过示例 |
|--------|----------|-----------|
| 身份声明 | 已认证或不声称医生身份 | 未认证却用"作为医生我建议" |
| 建议边界 | "建议咨询医生" | "你应该吃XX药" |
| 数据引用 | 标注来源（指南/文献） | "研究表明"但无出处 |
| 个体差异声明 | "因人而异""个体情况不同" | 暗示对所有人有效 |
| 风险提示 | 提及副作用和注意事项 | 只说好处不说风险 |

**《药品管理法》相关条款**：

| 条款 | 核心内容 |
|------|----------|
| 第89条 | 药品广告不得含功效断言；不得利用专家/患者名义推荐 |
| 第90条 | 药品广告内容不得与说明书不一致，需标明禁忌、不良反应 |
| 第91条 | 处方药只能在指定医学药学专业刊物发布广告 |

**《互联网广告管理办法》（2023年）**：

| 要点 | 内容 |
|------|------|
| 第7条 | 互联网广告应具有可识别性 |
| 第8条 | 禁止以健康科普等形式变相发布广告 |
| 第9条 | 体验分享形式推销商品需标明"广告" |
| 第10条 | 医疗药品广告需经审查机关批准 |

**"构成处方药广告"判定标准**：

同时满足以下2项即违规：
1. 可识别具体药品（品牌名/商品名/可明确指向的描述）
2. 存在推广意图（疗效宣传/购买引导/正面推荐之一）

高风险红线组合：
- ❌ 品牌名 + 疗效描述
- ❌ 品牌名 + 购买引导
- ❌ 品牌名 + 疗效 + 购买引导（三合一=铁定违规）
- ❌ 描述可唯一指向该药品 + 推广意图

#### 四、药品内容审核实操规则

**允许的内容形式**：

| 形式 | 条件 | 示例（根据实际药品替换） |
|------|------|------|
| 疾病科普 | 不指向特定品牌药 | "{适应症}的治疗原则" |
| 品类科普 | 只提药物类别不提品牌 | "{通用名}类药物的作用机制" |
| 用药教育 | 强调遵医嘱 | "服用{品类}期间注意事项" |
| 患者经历 | 不做推荐不提购买 | "我的{适应症}治疗经历（遵医嘱用药）" |
| 副作用管理 | 科普性质 | "用药期间{常见副作用}怎么办" |

**禁止的内容形式**：

| 形式 | 判定为 | 示例 |
|------|--------|------|
| 品牌名+效果好 | 处方药广告 | "{品牌名}真的有效" |
| 品牌名+购买方式 | 处方药广告+站外引流 | "XX药某宝有卖" |
| 伪装科普实为推广 | 变相广告 | 通篇科普最后引导某品牌 |
| 对比贬低竞品 | 不正当竞争 | "XX比YY好太多了" |
| 晒处方单/药盒 | 变相品牌露出 | 拍清晰药品包装 |
| 价格/渠道信息 | 购买引导 | "这个药XX元""在XX买" |

**违规话术 → 安全替代对照表（通用模板 + 策略报告补充）**：

以下为通用违规模式，审核时同时参照策略报告中「一、合规提醒」的 ❌→✅ 对照表作为补充：

| 违规表述模式 | 触犯条款 | 安全替代 |
|-------------|----------|----------|
| "{品牌名}+疗效描述" | 品牌+疗效 | "遵医嘱用药后{症状}有改善" |
| "X天/X周见效" | 疗效时间承诺 | "起效时间因人而异，需遵医嘱" |
| "比XX药好用多了" | 竞品贬低 | "是目前治疗方案之一" |
| "根治/治愈/永不复发" | 治愈承诺 | "有助于长期控制和改善" |
| "零副作用很安全" | 安全性断言 | "安全性总体良好，遵医嘱监测" |
| "不用去医院自己买就行" | 诱导不当用药 | "请在医生指导下使用" |
| "私我/滴滴/加V" | 站外引流 | "详情请咨询医生" |
| "一个疗程就能治好" | 疗程承诺 | "疗程因人而异，需医生评估" |
| "最好的药/第一/唯一" | 绝对化用语 | "是临床常用的治疗选择之一" |
| "所有人都适合" | 适用性断言 | "适用人群需医生评估，有禁忌症" |
| "纯天然没有副作用" | 虚假宣传 | "药物均有注意事项，请阅读说明书" |
| "医生都推荐这个" | 利用医生推荐 | "临床上有使用该类药物的方案" |
| 策略报告中的 ❌ 表述 | 参照策略合规提醒 | 对应的 ✅ 替代表述 |

**高风险关键词清单（动态构建）**：

审核时的关键词清单从以下来源动态组合：
- **品牌名/商品名**：从策略报告「一、药品基础档案」提取本品品牌名 + 从「竞品格局」提取竞品品牌名
- **疗效承诺词**（通用）：见效、治好、治愈、痊愈、根治、消除、去除、清零、搞定、战胜、告别、永久
- **时间承诺词**（通用）：X天、X周见效、一个疗程、速效、立刻、马上、当天
- **绝对化词**（通用）：最好、最强、最有效、第一、唯一、100%、绝对、肯定、一定、必须用
- **购买引导词**（通用）：买、购买、下单、链接、旗舰店、某宝、某东、私信问、评论区滴滴、主页有
- **策略专项禁区词**：从策略报告合规提醒中的 ❌ 表述提取关键词（如"根治便秘""排毒养颜""减肥瘦身"等非适应症暗示）

### Step 6：逐条审核

对每行已生成的笔记按以下维度检查：

**维度1：法律合规**

| 检查项 | 依据 | 严重程度 |
|--------|------|----------|
| 品牌名+疗效+购买引导三者共存 | 《广告法》第15条 | 高 |
| 疗效承诺/安全性断言 | 《广告法》第16条 | 高 |
| 绝对化用语 | 《广告法》第9条 | 高 |
| 虚构数据/未验证功效 | 《广告法》第28条 | 高 |
| 诱导不当用药 | 《药品管理法》第89条 | 高 |
| 变相广告（伪装科普） | 《互联网广告管理办法》第8条 | 高 |

**维度2：平台规范**

| 检查项 | 严重程度 |
|--------|----------|
| 引导站外交易 | 高 |
| 恶意引流/刷量互赞引导 | 高 |
| 低俗/擦边/暴力内容 | 高 |
| 标题党 | 中 |
| 未标注广告/合作 | 中 |
| 触发限流敏感词 | 中 |

**维度3：品牌露出合规**

| 检查项 | 合规线 | 修改方向 |
|--------|--------|----------|
| 品牌名出现次数 | ≤1次且不与疗效/购买同段 | 用品类代替 |
| 药品包装/价格展示 | 禁止 | 删除或模糊化 |
| 购买引导 | 禁止 | 引导"咨询医生" |
| 促销/折扣暗示 | 禁止 | 删除 |
| 竞品贬低 | 禁止 | "与传统方案相比" |

**维度4：疗效表述**

| 检查项 | 合规线 | 修改方向 |
|--------|--------|----------|
| 量化疗效承诺 | 禁止 | "临床研究显示有改善" |
| 治愈性承诺 | 禁止 | "改善""缓解""管理" |
| 暗示无需配合其他措施 | 禁止 | 加"配合饮食运动" |
| 安全性绝对化 | 禁止 | "安全性良好""耐受性好" |
| 适用性断言 | 禁止 | "适用人群需医生评估" |

**维度5：高风险关键词扫描**

对正文逐词扫描品牌名/疗效承诺/时间承诺/绝对化/购买引导词库，命中即标记并判断上下文。

**维度6：必备要素检查**

| 检查项 | 要求 | 缺失处理 |
|--------|------|----------|
| "遵医嘱"/"请在医生指导下使用" | 必须存在 | 修正版末尾补充 |
| 正文字数 | 小红书300-800 / 抖音200-400 | 超长精简/过短标注 |
| 标签与内容相关性 | 匹配 | 建议替换 |
| 个体差异声明 | 建议有 | 建议补充 |

**维度7：内容质量（辅助，不扣合规分）**

| 检查项 | 影响 |
|--------|------|
| 开头吸引力 | 完读率 |
| 互动引导 | 互动率 |
| 结构清晰度 | 收藏率 |
| 是否偏离分配角度 | 内容去重 |

### Step 7：生成修正版

对审核不通过（⚠️或❌）的笔记：
1. 对照「违规话术→安全替代对照表」逐项替换
2. 保持原有人设风格和内容角度不变
3. 在审核结果中标注每处具体改了什么

### Step 8：写回表格

在原表基础上新增以下列：

| 列号 | 字段名 | 内容 |
|------|--------|------|
| 23 | 分配角度 | 该达人被分配的内容切入角度 |
| 24 | 笔记标题 | 生成的标题（20字以内，含emoji） |
| 25 | 笔记正文 | 生成的完整正文 |
| 26 | 话题标签 | #xx #xx 格式 |
| 27 | 封面建议 | 封面构图方向 |
| 28 | 合规审核结果 | 结构化审核报告（见下方格式） |
| 29 | 修改后正文 | 审核不通过时提供修正版；通过则填"无需修改" |

写回规则：
- 不修改原有22列的任何数据
- 按行对应
- 输出格式与输入一致（xlsx输入→xlsx输出，csv输入→csv输出）
- xlsx：单元格内换行使用实际换行符
- csv：含换行或逗号的字段用双引号包裹，编码与输入文件一致

**第28列格式**：
```
【结论】✅通过 / ⚠️需修改 / ❌不建议发布
【评分】X/10
【问题】
1. [严重程度] 问题描述 → 修改建议
2. ...
【风险】封号:低/中/高 | 限流:低/中/高
```

## 评分标准

| 分数 | 含义 | 对应结论 |
|------|------|----------|
| 9-10 | 完全合规，内容质量好 | ✅通过 |
| 7-8 | 基本合规，有小瑕疵可优化 | ✅通过（附建议） |
| 5-6 | 存在需修改的问题 | ⚠️需修改 |
| 3-4 | 多处违规，需大幅修改 | ⚠️需修改 |
| 1-2 | 严重违规，建议重写 | ❌不建议发布 |

扣分规则：
- 高严重程度：每项 -2~3 分
- 中严重程度：每项 -1~2 分
- 低严重程度：每项 -0.5~1 分
- 缺失"遵医嘱"：直接 -2 分
- 命中高风险关键词且构成违规：每个 -1~2 分
