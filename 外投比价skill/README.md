# 外投 PO 报价核验 Skill

一个用于外投达人/博主采购场景的报价核验Skill。它可以读取供应商 PO 报价单、结算单和报价基准库，自动比对供应商报价是否合理，并输出可追溯的 Markdown 核验报告。

该 Skill 的核心目标是帮助采购、投放、商务和审核同学快速判断：

- 供应商 PO 报价是否高于平台或历史基准价；
- 结算单价是否与 PO 单价一致；
- 达人报价是否存在倒挂、严重低报、口径异常或缺少可比数据；
- CPM / CPE 等效果指标是否需要进一步人工复核；
- 报价基准来自在线库、离线导出文件，还是同类样本估算。

## Skill 名称

```yaml
name: external-quote-audit
```

## 适用场景

当你需要审核以下文件或问题时，可以使用该 Skill：

- 供应商 PO 报价单；
- 供应商结算单；
- 外投达人报价清单；
- 小红书蒲公英报价库；
- JOYspace 在线报价库；
- 达人报价合理性判断；
- 结算价与 PO 价一致性核验；
- CPM / CPE 性价比核验；
- 缺少精确匹配时的同类样本估算。

示例提问：

```text
请使用 external-quote-audit 核验这份供应商 PO 报价单，判断报价是否合理。
```

```text
请比对结算单和 PO 单，检查结算单价是否超出 PO 或蒲公英平台价。
```

```text
这批达人没有完整蒲公英链接，请基于同类账号样本估算报价是否偏高。
```

## 核心能力

### 1. 在线报价库优先

Skill 默认优先查询 JOYspace 在线报价库：

```text
https://joyspace.jd.com/sheets/zLYydbGAzSAEUNscQaL3?themeColor=maxdeep&env=timline&lang=zh_CN
```

如果在线文档无法访问，例如需要登录、没有权限或网络不可达，Skill 会要求用户提供替代数据：

- 最新导出的 Excel 报价库；
- 复制出来的表格数据；
- 截图；
- 可访问在线文档的账号或权限说明。

> 注意：如果没有实际访问到在线库，报告中必须明确说明“未读取到最新在线数据”，不得假装已查询最新报价库。

### 2. 支持多种输入

Skill 可处理以下一种或多种文件：

- PO 报价单；
- 结算单；
- 蒲公英平台报价库；
- 自定义报价基准表；
- 包含达人名称、主页链接、蒲公英链接、粉丝数、报价、CPM/CPE 的投放数据表。

### 3. 多级匹配逻辑

Skill 会按以下优先级匹配达人或账号：

1. 达人 ID / 小红书 ID 精确匹配；
2. 主页链接或蒲公英链接精确匹配；
3. 达人名称精确匹配；
4. 单一高置信度模糊匹配；
5. 同类样本估算。

若出现多个可能匹配对象，会标记为：

```text
⚠️ 需人工确认匹配
```

### 4. 图文 / 视频价格口径区分

对于小红书蒲公英报价库，Skill 会根据内容形式选择对应价格：

- 图文内容：使用 `图文笔记一口价`；
- 视频内容：使用 `视频笔记一口价`。

若内容形式缺失，但图文和视频价格同时存在，则不会直接给出通过结论，而是标记为需要确认内容形式。

### 5. 风险判定

报价比值计算方式：

```text
报价比值 = 供应商报价 / 基准价格
```

默认判定规则：

| 报价比值 | 判定 |
|---:|---|
| > 100% | 🔴 倒挂，供应商报价高于基准价 |
| 80% – 100% | ✅ 合理 |
| 50% – 80% | ⚠️ 低价需核实 |
| < 50% | 🔴 严重低报 / 口径异常 |
| 无可靠基准 | ⚠️ 无可比数据 |

### 6. 结算稽核

当同时提供结算单和 PO 单时，Skill 会检查：

- 结算单价是否高于 PO 单价；
- 结算单价是否低于 PO 单价；
- 是否存在有结算无 PO 的记录；
- 结算价是否超过蒲公英或外部参考价。

### 7. CPM / CPE 核验

Skill 会优先读取文件中已有的 CPM / CPE。若缺失但有价格、曝光、播放或互动数据，则可按以下方式计算：

```text
CPM = 价格 / 曝光或播放量 × 1000
CPE = 价格 / 互动量
```

如果缺少足够数据，Skill 会标记为：

```text
⚠️ 无 CPM/CPE 基准数据
```

## 输出报告

Skill 默认输出 Markdown 格式报告，包含：

- 场景类型；
- 使用文件；
- 报价库来源；
- 查询或读取时间；
- 使用的数据库 sheet；
- 核验总数；
- 精确匹配数；
- 需人工确认匹配数；
- 无可比数据数；
- 综合风险等级；
- 核验明细表；
- 异常与待跟进事项；
- 建议动作；
- 关键假设。

明细表字段通常包括：

| 达人/账号 | 平台 | 内容形式 | 匹配依据 | 基准来源 | 供应商报价/结算价 | 基准价格 | 报价比值 | CPM | CPE | 风险 | 说明 |
|---|---|---|---|---|---:|---:|---:|---:|---:|---|---|

## 数据源说明

Skill 会区分以下数据来源：

- `JOYspace 在线文档`：优先数据源；
- `离线导出 Excel`：在线文档不可访问时使用；
- `截图 / 复制表格`：临时补充数据；
- `用户自定义基准表`：优先于系统推断；
- `同类样本估算`：无账号级精确匹配时的辅助判断。

> 重要：外部参考价不能被称为“蒲公英平台价”。报告中必须标明每个基准价格来自哪个 sheet 和哪个字段。

## 安装方式

### 安装到 Codex

将本 Skill 文件夹放入 Codex skills 目录：

```text

```

```powershell
Copy-Item -Recurse .\external-quote-audit C:\Users\<your-user>\.codex\skills\external-quote-audit
```

安装后重启 Codex，使新 Skill 生效。

将 `SKILL.md` 作为工具说明或 agent skill 文档引入，并确保相关引用文件（如 `references/database-schema.md`）一并放入同一目录结构。

推荐目录结构：

```text
external-quote-audit/
├── SKILL.md
├── README.md
└── references/
    └── database-schema.md
```

## 使用示例

### 供应商 PO 报价核验

```text
请使用 external-quote-audit 核验这份供应商 PO 报价单。
优先查询 JOYspace 在线报价库；如果无法访问，请使用我上传的离线报价库。
输出报价合理性、风险等级和需人工跟进事项。
```

### 结算单 + PO 单稽核

```text
请核验这份结算单和对应 PO 单：
1. 检查结算单价是否与 PO 单一致；
2. 检查是否超过蒲公英平台价；
3. 标记所有异常记录；
4. 输出 Markdown 核验报告。
```

### 无精确匹配时估算

```text
这批达人没有完整蒲公英链接。
请先按名称和主页链接匹配；无法精确匹配的，基于同平台、同内容形式、相似粉丝量级做同类样本估算。
```

## 注意事项

- 不要编造数据库中不存在的价格或行业区间；
- 不要把抖音或其他外部参考价写成蒲公英平台价；
- 没有访问到 JOYspace 在线库时，必须在报告中说明；
- 内容形式不明时，不要直接给出“通过”结论；
- 不要只按粉丝量粗暴比价，优先做账号级匹配；
- 不要自动合并不同平台的同名账号；
- 所有假设、缺失字段和匹配失败都必须披露。
